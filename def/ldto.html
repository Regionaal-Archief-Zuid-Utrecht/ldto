Description

  • De RDF-ontologie van LDTO (Linked-data Duurzaam Toegankelijke Overheidsinformatie)

  • The RDF ontology of LDTO (Linked Data Sustainable Accessible Government Information). This Ontology is used by RAZU describe digital archival records long term preservation and access. Its core is based on MDTO, the National Archives'standard for recording and exchanging metadata, referenced here via skos:closeMatch property. Other ontologies and vocabularies, such as schema and premis, are reused to describe domain-specific content related information. Note that LDTO will be extended progressively to describe new types of records ingested.

Information object. c

IRI https://data.razu.nl/def/ldto/Informatieobject
Description

A self-contained set of data with its own identity. An information object (IO) can be composed of smaller information objects. This is called an aggregation. For example, the file of a permit application is an aggregation, containing the application, decision and all other relevant IOs. IOs can be represented by ldto:Bestand entities, ex: a .doc file or a .pdf representation of an IO.

Scope Note

Information objects in the context of LDTO are information objects that arise from government tasks.
Example 
A document, a database, a dossier, a website.

Coverage in time. op

IRI https://data.razu.nl/def/ldto/dekkingInTijd
Description

A point in time or period to which the content of the information object relates.
Scope Note

In LDTO, the property and its related metadata, refer to the relative time coverage of the records stored at RAZU, not to the absolute time covered by those works. For instance, a publication may have been issued from 1930 to 1980, but RAZU might store only records issued from 1950 to 1980. More details should be specified via ldto:dekkingInTijdType.

Events described using these metadata should be those happened to an IO before its ingestion at RAZU, hence,these information should be submitted, if known, by archive creators. Events happening to IO during transfer and ingestion at RAZU will be recorded in a separate event log.
